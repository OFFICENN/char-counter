# 文字カウンターアプリ 作成手順書

## 概要

Cursor + Vite + React + TailwindCSS で文字カウンターWebアプリを作成し、
GitHub + Vercel で公開するまでの手順をまとめたものです。

---

## 使用ツール

| ツール | 用途 |
|---|---|
| Cursor | コード作成・編集（AI搭載IDE） |
| Vite | 開発サーバー・ビルドツール |
| React + TypeScript | UIフレームワーク |
| TailwindCSS v4 | スタイリング |
| GitHub | ソースコード管理 |
| Vercel | ホスティング・自動デプロイ |

---

## STEP 1：プロジェクト作成

```bash
npm create vite@latest char-counter -- --template react-ts
cd char-counter
npm install
```

## STEP 2：TailwindCSS インストール

```bash
npm install -D tailwindcss @tailwindcss/vite
```

### vite.config.ts に追加

```ts
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";
import tailwindcss from "@tailwindcss/vite";

export default defineConfig({
  plugins: [react(), tailwindcss()],
});
```

### src/index.css の先頭に追加

```css
@import "tailwindcss";
```

## STEP 4：動作確認

```bash
npm run dev
# → http://localhost:5173/ で確認
```

## STEP 5：GitHubにリポジトリ作成 & Push

### 5-1. GitHubでリポジトリ作成

1. https://github.com/new を開く
2. Repository name: `char-counter`
3. Public を選択
4. README等のチェックはすべて外す
5. 「Create repository」をクリック

### 5-2. ローカルからPush

```bash
git init
git add -A
git commit -m "feat: 文字カウンターWebアプリ初期実装"
git remote add origin https://github.com/あなたのユーザー名/char-counter.git
git branch -M main
git push -u origin main
```

## STEP 6：Vercelでデプロイ

1. https://vercel.com/ にGitHubアカウントでログイン
2. 「Add New Project」をクリック
3. `char-counter` リポジトリを選択して「Import」
4. 設定はデフォルトのまま「Deploy」をクリック
5. 1〜2分で公開完了
6. 公開URL（例: `https://char-counter-xxxxx.vercel.app`）が発行される

## STEP 7：以降の更新方法

コードを修正したら以下を実行するだけで自動デプロイされる：

```bash
git add -A
git commit -m "変更内容の説明"
git push
```

Vercelが自動検知して再ビルド・再デプロイしてくれる。

## 実装済み機能一覧

- リアルタイム文字数カウント（スペース有/無・単語数・行数・段落数）
- 文字数制限プリセット（Twitter / Instagram / note / メタディスクリプション / ブログタイトル / カスタム）
- プログレスバー（80%超で黄色、超過で赤）
- テキストのコピー & クリア
- ダークモード（システム設定追従 + 手動トグル）
- ユーザー名カスタマイズ（環境変数固定 / URLパラメータ / モーダル設定）
- レスポンシブ対応（PC・スマホ）

---

## 補足：主要な技術ポイント

- TailwindCSS v4 は `@tailwindcss/vite` プラグインで統合（v3以前とは設定方法が異なる）
- ダークモードは `class` ベース（HTMLのルートに `dark` クラスを付与）
- 環境変数は `VITE_` プレフィックスが必須（Viteの仕様）
- `vercel.json` でビルドコマンドと出力ディレクトリを明示指定
